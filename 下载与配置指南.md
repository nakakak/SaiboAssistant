# 赛搏小小助手（SaiboAssistant）v0.2.11 — 下载、配置与使用指南

本文仅说明 **v0.2.11** 这一版 Release 中 **`SaiboAssistant-*` 单文件** 的下载与配置流程。  
请下载对应系统的单文件后直接运行，**无需解压**；首次运行会在程序旁自动生成 `config.yaml` 并进入图形向导。

---

## 二、下载后如何启动

1. 将下载的 **单个文件** 放到任意目录（如 `~/SaiboAssistant/` 或桌面）。
2. **Mac / Linux**（若尚未可执行）：
   ```bash
   chmod +x SaiboAssistant-macOS-arm64   # 文件名按实际修改
   ./SaiboAssistant-macOS-arm64
   ```
3. **Windows**：双击 `SaiboAssistant-Windows-x64.exe`。

**首次运行：**

- 在 **可执行文件同目录** 自动生成 `config.yaml`（目录不可写时，写入系统用户配置目录下的 `SaiboAssistant/config.yaml`）。
- 弹出 **「赛搏小小助手 — 首次配置」** 向导（3 步：功能选择 → 连接信息 → 权限与同意）。

---

## 三、首次配置向导（界面说明）

### 第 1 步：欢迎与功能选择

- **OpenClaw：云端任务发到本电脑** — 设备经商家云把任务派到本机 OpenClaw。
- **听写：语音转文字传到本电脑** — 听写内容经云端转发到本机（可注入输入框）。

**至少勾选一项**，点 **「下一步」**。

### 第 2 步：商家云与设备配对（必填）

#### （1）商家云地址

须包含路径 `/bridge/connector`，**不要**手写 `?token=` 等 query。

```text
ws://127.0.0.1:8084/bridge/connector
wss://你的域名/bridge/connector
```

#### （2）连接码 与 设备 MAC（二选一）

| 方式 | 填写内容 |
|------|----------|
| **连接码（6 位数字）**（默认） | 如 `123456` |
| **设备 MAC** | 如 `98:88:e0:13:85:38` |

只填连接码时，程序会向商家云自动解析 MAC 并写回 `config.yaml`。

#### （3）OpenClaw（仅勾选 OpenClaw 时显示）

- **连接 OpenClaw 处理任务**：勾选后连真实 Gateway；不勾选则为 **echo 自测**。
- **Gateway Token**：见下文第六节。
- **OpenClaw 地址**：本机常用 `http://127.0.0.1:18789`。

点 **「下一步」**。

### 第 3 步：权限与同意

勾选两项同意；macOS 听写注入需在 **辅助功能** 中允许本程序。点 **「完成配置」**。

## 四、配置完成后：主界面与日常使用

### 4.1 运行状态窗口

- 显示商家云地址与连接状态（**已连接商家云** 表示成功）。
- **连接设置** / **重连云端** / **退出程序**。
- 系统托盘：运行状态、连接设置、通道设置、重连、退出。

### 4.2 修改配置

托盘或运行状态窗 → **连接设置** → **保存并重连**。  
`config.yaml` 在程序同目录（或系统配置目录）。

## 五、如何获取设备 MAC 地址

### 方式 A：6 位连接码（推荐）

1. 在商家云管理台为设备生成 **电脑连接码**，或从设备端查看/听取播报。
2. 向导中选 **连接码**，填入 6 位数字。

程序会调用：

```http
POST https://你的商家云域名/api/public/connector-code-resolve
{"code": "123456"}
```

**管理台生成连接码（需登录）：**

```bash
curl -X POST "https://你的商家云域名/api/device/connector-pair-code" \
  -H "Authorization: Bearer <你的用户JWT>" \
  -H "Content-Type: application/json" \
  -d '{"device_mac":"98:88:e0:13:85:38"}'
```

### 方式 B：直接填 MAC

从管理台设备列表、机身标贴或烧录日志查看，格式如 `98:88:e0:13:85:38`。

## 六、如何获取 OpenClaw Gateway Token

> 与商家云连接码 **不是同一项**；仅在使用真实 OpenClaw（`gateway_ws`）时需要。

**macOS / Linux：**

```bash
jq -r '.gateway.auth.token' ~/.openclaw/openclaw.json
```

**Windows（PowerShell）：**

```powershell
(Get-Content "$env:USERPROFILE\.openclaw\openclaw.json" | ConvertFrom-Json).gateway.auth.token
```

启动 Gateway（示例，以本机 OpenClaw 为准）：

```bash
openclaw gateway
```

再填 **OpenClaw 地址**（如 `http://127.0.0.1:18789`）与 **Gateway Token**。

## 八、配置完成后如何使用

1. **OpenClaw 任务**：助手已连云、OpenClaw 通道已开、Gateway 已运行；设备绑定正确后即可派发任务。
2. **听写**：开启听写通道；设备上「开始传输」；注入模式需先点好 PC 输入框。
3. **保持在线**：程序需常驻；退出后设备侧显示 Connector 离线。

## 十、常见问题

| 现象 | 处理 |
|------|------|
| Mac 无法打开 | 系统设置 → 仍要打开；或 `xattr -cr ./SaiboAssistant-macOS-arm64` |
| 无法连接商家云 | 检查 `server_url`、端口、`bridge_enabled` |
| 连接码无效 | 确认 6 位、未过期，重新生成 |
| OpenClaw 失败 | Gateway 是否运行、Token 是否与 `openclaw.json` 一致 |
| macOS 听写不注入 | 辅助功能中允许本程序 |
