# 赛搏小小助手 — 连接使用指南

让 **喵伴** 和您电脑上的 **OpenClaw（小龙虾）** 连起来，按下面步骤做即可。

---

## 第一步：下载并打开程序

按您的电脑 **直接点击下表链接** 下载（当前版本 **v0.2.19**）：

| 您的电脑 | 点击下载 |
|----------|----------|
| Mac（M 系列芯片） | [SaiboAssistant-macOS-arm64](http://gitlab.cyberai.top/gitlab/show/saiboassistant/-/raw/release-v0.2.19/releases/v0.2.19/SaiboAssistant-macOS-arm64) |
| Mac（Intel） | [SaiboAssistant-macOS-x64](http://gitlab.cyberai.top/gitlab/show/saiboassistant/-/raw/release-v0.2.19/releases/v0.2.19/SaiboAssistant-macOS-x64) |
| Windows | [SaiboAssistant-Windows-x64.exe](http://gitlab.cyberai.top/gitlab/show/saiboassistant/-/raw/release-v0.2.19/releases/v0.2.19/SaiboAssistant-Windows-x64.exe) |
| Linux x64 | [SaiboAssistant-Linux-x64](http://gitlab.cyberai.top/gitlab/show/saiboassistant/-/raw/release-v0.2.19/releases/v0.2.19/SaiboAssistant-Linux-x64) |
| Linux arm64 | [SaiboAssistant-Linux-arm64](http://gitlab.cyberai.top/gitlab/show/saiboassistant/-/raw/release-v0.2.19/releases/v0.2.19/SaiboAssistant-Linux-arm64) |

项目主页（查看本说明与所有安装包）：  
http://gitlab.cyberai.top/gitlab/show/saiboassistant/-/tree/release-v0.2.19

Mac 用户下载后，终端执行一次：

```bash
chmod +x SaiboAssistant-macOS-arm64
```

若提示「无法验证开发者」：系统设置 → 隐私与安全性 → **仍要打开**

然后 **双击运行** 赛搏小小助手，菜单栏/托盘会出现图标。

---

## 第二步：电脑上先准备好两样东西

| 准备项 | 怎么做 |
|--------|--------|
| **OpenClaw 网关**（要用龙虾对话时） | 打开终端，输入 `openclaw gateway`，保持窗口不要关 |
| **Gateway 令牌** | 终端输入：`jq -r '.gateway.auth.token' ~/.openclaw/openclaw.json`，复制出来的字符串备用 |

> 只用「听写转发」、不用 OpenClaw 时，可跳过 OpenClaw 相关项。

---

## 第三步：在喵伴上看配对码

1. 喵伴连上 WiFi，能正常使用。
2. **先退出** 电脑上的赛搏小小助手（否则屏幕可能不显示码）。
3. 喵伴：**圆环菜单 → OpenClaw 对话**（或 **听写转发** 页）。
4. 屏幕上会显示 **6 位配对码**（例如 `537724`），记下来。

> 配对码一般 **长期不变**，填过一次后通常不用重复填。  
> 也可不用配对码，改在 [赛搏 AI 管理台](https://xiaozhi.cyberai.top) → **设备管理** 里复制 **MAC 地址**。

---

## 第四步：在助手里完成首次配置

首次运行会弹出 **「首次配置」** 向导，按界面操作：

1. **选功能**：至少勾选一项  
   - OpenClaw：和龙虾对话  
   - 听写：语音转文字到电脑
2. **连商家云**（一般不用改地址，默认已是公网）：  
   `wss://xiaozhi.cyberai.top/bridge/connector`
3. **绑定设备**（二选一）：  
   - **推荐**：选「连接码」，填入喵伴屏幕上的 **6 位数字**  
   - 或选「设备 MAC」，粘贴管理台里的 MAC
4. **OpenClaw**（若已勾选）：  
   - 勾选「连接 OpenClaw 处理任务」  
   - **OpenClaw 地址**：`http://127.0.0.1:18789`  
   - **Gateway Token**：粘贴第二步复制的令牌
5. **完成** → 点「完成配置」。

**macOS 听写**：系统设置 → 隐私与安全性 → **辅助功能** → 允许「赛搏小小助手」。

---

## 第五步：确认已连上

| 看哪里 | 正常表现 |
|--------|----------|
| 助手托盘 / 运行状态 | 显示 **「已连接商家云」** |
| 喵伴 OpenClaw 页 | **配对码消失**，可以开始对话 |
| 说话测试 | 喵伴 OpenClaw 页点 **开始对话**，说完应有回复和声音 |

配置完成后窗口可能自动收起，程序在 **菜单栏（Mac）或托盘** 里继续运行，这是正常的。

## 日常使用

### OpenClaw 对话

1. 电脑：`openclaw gateway` 保持运行，赛搏小小助手保持在线。  
2. 喵伴：进入 **OpenClaw 对话** → **开始对话** → 说话即可。

### 听写转发（把话写到电脑输入框）

1. 电脑上先 **点一下** 要输入文字的位置（如微信、备忘录）。  
2. 喵伴：**听写转发** → **开始传输** → 说话。  
3. 识别文字会出现在电脑当前输入框。

### 以后要改设置

托盘图标 → **连接设置** → 改完后点 **保存并重连**。

| 想做什么 | 操作 |
|----------|------|
| 换一台喵伴 | 连接设置里重新填 **配对码** 或 **MAC** |
| 重走一遍向导 | 把同目录下 `config.yaml` 里的 `first_run` 改为 `true`，重启程序 |
| 查看是否在运行 | 托盘 → **运行状态** |

## 常见问题

| 现象 | 怎么办 |
|------|--------|
| 喵伴上看不到配对码 | 先 **完全退出** 赛搏小小助手，再进 OpenClaw 页查看 |
| 助手显示未连接 | 检查网络；托盘 → 连接设置 → **保存并重连** |
| OpenClaw 没反应 | 确认 `openclaw gateway` 在运行，Token 是否填对 |
| Windows 听写不出字 | 升级到 **v0.2.18+**；使用前先在电脑上点一下目标输入框，再点喵伴「开始传输」 |
| Mac 听写不出字 | 系统设置 → 辅助功能 → 允许本程序 |
| Mac 打不开程序 | `xattr -cr ./SaiboAssistant-macOS-arm64` 后再双击 |
