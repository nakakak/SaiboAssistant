# 赛搏小小助手（SaiboAssistant）— 下载、配置与使用指南

本仓库 **仅提供 Release 安装包**，不含源码。请按本文从下载到连接商家云、OpenClaw 的完整流程操作。

---

## 一、从哪里下载

### 1.1 Release 页面（推荐）

打开最新版本页面：

**https://github.com/nakakak/SaiboAssistant/releases/latest**

在 **Assets（附件）** 区域，按你的电脑选择 **一个** 文件下载（不要点 *Source code*，那是 GitHub 自动打的源码包，不是安装程序）。

| 你的电脑 | 下载文件名 |
|----------|------------|
| Windows 64 位 | `SaiboAssistant-Windows-x64.exe` |
| Mac Apple Silicon（M 系列） | `SaiboAssistant-macOS-arm64` |
| Mac Intel | `SaiboAssistant-macOS-x64` |
| Linux x86_64 | `SaiboAssistant-Linux-x64` |
| Linux arm64 | `SaiboAssistant-Linux-arm64` |

若需要附带 `README`、`config.example.yaml` 的完整目录包，可改下对应平台的 `openclaw-connector_v版本号_平台_架构.tar.gz`（或 Windows 的 `.zip`）。**单文件与压缩包内的程序功能相同**，一般用户只需下载上表中的单文件即可。

### 1.2 固定直链（始终指向最新版）

将下面链接中的仓库地址换成你的 Release 仓库即可；当前为：

```text
https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-Windows-x64.exe
https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-macOS-arm64
https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-macOS-x64
https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-Linux-x64
https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-Linux-arm64
```

### 1.3 命令行下载示例

**macOS Apple Silicon：**

```bash
curl -L -o SaiboAssistant-macOS-arm64 \
  https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-macOS-arm64
chmod +x SaiboAssistant-macOS-arm64
```

**Windows（PowerShell）：**

```powershell
Invoke-WebRequest -Uri "https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-Windows-x64.exe" -OutFile "SaiboAssistant-Windows-x64.exe"
```

**Linux x64：**

```bash
curl -L -o SaiboAssistant-Linux-x64 \
  https://github.com/nakakak/SaiboAssistant/releases/latest/download/SaiboAssistant-Linux-x64
chmod +x SaiboAssistant-Linux-x64
```

下载完成后可用 Release 附件中的 `checksums-sha256.txt` 校验文件完整性（可选）：

```bash
shasum -a 256 SaiboAssistant-macOS-arm64
# 与 checksums-sha256.txt 中对应行比对
```

## 二、下载后如何启动

### 2.1 放置与运行

1. 将下载的文件放到任意目录（例如 `~/SaiboAssistant/` 或桌面）。
2. **Mac / Linux**：首次需赋予执行权限（若已 `chmod +x` 可跳过）：
   ```bash
   chmod +x SaiboAssistant-macOS-arm64   # 文件名按实际修改
   ./SaiboAssistant-macOS-arm64
   ```
3. **Windows**：双击 `SaiboAssistant-Windows-x64.exe` 即可。

### 2.2 首次运行会发生什么

- 程序会在 **可执行文件同目录** 自动生成 `config.yaml`（若该目录不可写，则写入系统用户配置目录下的 `SaiboAssistant/config.yaml`）。
- 若尚未完成配置，会弹出 **「赛搏小小助手 — 首次配置」** 图形向导（约 620×620 窗口）。
- 向导共 **3 步**：功能选择 → 连接信息 → 权限与同意 → **完成配置**。

关闭向导而不保存时，下次启动仍会进入向导。

## 三、首次配置向导（界面说明）

### 第 1 步：欢迎与功能选择

界面内容：

- 应用名称：**赛搏小小助手**
- 两个可勾选项（**至少选一项**）：
  - **OpenClaw：云端任务发到本电脑** — 设备经商家云把任务派到本机 OpenClaw 执行。
  - **听写：语音转文字传到本电脑** — 设备听写内容经云端转发到本机（可注入输入框）。

勾选后点 **「下一步」**。

### 第 2 步：商家云与设备配对（必填）

#### （1）商家云地址

填写 **WebSocket 地址**，必须包含路径 `/bridge/connector`，**不要**自己加 `?token=` 等参数。

示例：

```text
ws://127.0.0.1:8084/bridge/connector          # 本机联调
wss://你的域名/bridge/connector               # 线上 HTTPS 环境
```

向商家或运维索取实际地址与端口。

#### （2）设备标识：连接码 与 MAC **二选一**

界面提供单选：

| 方式 | 适用场景 | 填写内容 |
|------|----------|----------|
| **连接码（6 位数字）**（默认） | 不知道 MAC，或设备/管理台已给出连接码 | 6 位数字，如 `123456` |
| **设备 MAC** | 已知设备 MAC | 如 `98:88:e0:13:85:38`（冒号分隔） |

**只填连接码时**：保存并启动后，程序会向商家云 API 自动解析出 MAC，并写回 `config.yaml`，无需手工查 MAC。

#### （3）OpenClaw（仅当第 1 步勾选了 OpenClaw 时显示）

- **连接 OpenClaw 处理任务**：勾选后使用真实 OpenClaw；不勾选则使用 **echo 自测模式**（只回显文字，不连 OpenClaw）。
- **Gateway Token**：本机 OpenClaw 网关令牌（见下文「如何获取 OpenClaw Token」）。
- **OpenClaw 地址**：Gateway 的 HTTP 基址，本机一般为 `http://127.0.0.1:18789`；OpenClaw 在别的机器上则填对应地址。
- **更多 OpenClaw 选项**（可展开）：`gateway_ws_url`、`model` 等，一般可留空。

填写完成后点 **「下一步」**。

### 第 3 步：权限与同意

- 阅读 **权限与网络**、**自愿使用** 说明。
- 勾选两项同意。
- **macOS** 若开启听写注入，可点 **「打开系统隐私设置（macOS 辅助功能）」**，在「隐私与安全性 → 辅助功能」中允许本程序。
- 点 **「完成配置」** 保存并进入主界面。

## 四、配置完成后：主界面与日常使用

### 4.1 运行状态窗口

向导完成后会出现 **「赛搏小小助手 — 运行状态」** 窗口，典型显示：

| 区域 | 含义 |
|------|------|
| 商家云地址 | 当前 `server_url` |
| 状态行 | 如「正在连接商家云…」→「**已连接商家云**」 |
| 详情 | 「运行中，可接收云端任务与听写」 |
| **连接设置** | 修改商家云、MAC/连接码、OpenClaw、听写等 |
| **重连云端** | 断线或改配置后手动重连 |
| **退出程序** | 完全退出（关窗口只是隐藏） |

**macOS / Windows** 菜单栏或系统托盘会有图标，右键菜单包括：运行状态、连接设置、通道设置、重连、退出等。

### 4.2 连接成功的标志

- 运行状态显示 **「已连接商家云」**。
- 终端日志（若从命令行启动）可见：`connector: connected ws=...`
- 设备侧发起 OpenClaw 任务或听写时，本机应有相应日志或注入/执行结果。

### 4.3 后续修改配置

不必重走向导，随时：

1. 托盘 → **连接设置**，或运行状态窗 → **连接设置**。
2. 修改后点 **「保存并重连」**。

`config.yaml` 与程序同目录（或系统配置目录），也可用文本编辑器修改，保存后点 **重连云端**。

## 五、如何获取设备 MAC 地址

设备 MAC 是喵伴/设备在商家云上的唯一标识（格式如 `98:88:e0:13:85:38`）。任选一种方式即可。

### 方式 A：使用 6 位连接码（推荐，不必先查 MAC）

1. 在 **商家云管理台 / 设备管理** 中，为对应设备生成 **电脑连接码（Connector 配对码）**（6 位数字）。
2. 或在设备端进入相关设置页，查看或语音播报的连接码。
3. 在助手向导中选 **「连接码（6 位数字）」**，填入即可。

程序启动时会请求：

```http
POST https://你的商家云域名/api/public/connector-code-resolve
Content-Type: application/json

{"code": "123456"}
```

成功后会返回 `device_mac` 并写入本地配置。

**管理台 API 生成连接码（需登录用户 Token）：**

```bash
curl -X POST "https://你的商家云域名/api/device/connector-pair-code" \
  -H "Authorization: Bearer <你的用户JWT>" \
  -H "Content-Type: application/json" \
  -d '{"device_mac":"98:88:e0:13:85:38"}'
```

返回示例：

```json
{
  "status": "ok",
  "data": {
    "code": "123456",
    "expires_at": 1716451200
  }
}
```

将 `code` 填入助手即可。连接码有时效，过期需重新生成。

### 方式 B：直接填写 MAC

在以下位置查看 MAC 后，在向导中选 **「设备 MAC」** 填写：

| 来源 | 说明 |
|------|------|
| 商家云管理台 | 设备列表中的 MAC / 设备 ID |
| 设备包装盒或机身标贴 | 印刷的 MAC |
| 设备串口 / 烧录日志 | 如 `MAC: 98:88:e0:13:85:38` |
| 路由器已连接设备列表 | 仅作参考，以管理台为准 |

**格式**：建议大写或小写均可，冒号分隔，如 `98:88:e0:13:85:38`。

## 六、如何获取 OpenClaw Gateway Token

> **重要**：这是 **OpenClaw 本机网关** 的令牌，与商家云 MCP token、连接码 **不是同一个东西**。  
> 仅在勾选「连接 OpenClaw 处理任务」且 `openclaw.mode` 为 `gateway_ws` 时需要。

### 6.1 从配置文件读取（最常用）

OpenClaw 配置一般在用户目录下的 `openclaw.json`：

**macOS / Linux：**

```bash
# 查看配置文件路径（若存在）
ls -la ~/.openclaw/openclaw.json

# 提取 gateway.auth.token
jq -r '.gateway.auth.token' ~/.openclaw/openclaw.json
```

若无 `jq`，可用 Python：

```bash
python3 -c "import json; print(json.load(open('$HOME/.openclaw/openclaw.json'))['gateway']['auth']['token'])"
```

**Windows（PowerShell）：**

```powershell
Get-Content "$env:USERPROFILE\.openclaw\openclaw.json" | ConvertFrom-Json | Select-Object -ExpandProperty gateway | Select-Object -ExpandProperty auth | Select-Object -ExpandProperty token
```

将输出的字符串复制到助手 **「Gateway Token」** 或配置项 `openclaw.gateway_token`。

### 6.2 确认 OpenClaw Gateway 已启动

本机联调时，需先在本机或目标服务器启动 OpenClaw Gateway（默认 WebSocket 端口 **18789**）。具体命令以你安装的 OpenClaw 文档为准，常见为：

```bash
# 示例（以你本机 OpenClaw 安装方式为准）
openclaw gateway
# 或
openclaw gateway run
```

确认 Gateway 监听后再填：

- **OpenClaw 地址**：`http://127.0.0.1:18789`（本机）或 `http://服务器IP:18789`
- **Gateway Token**：上一步读取的值

### 6.3 仅做链路自测（不连 OpenClaw）

若暂时没有 OpenClaw，可 **不勾选**「连接 OpenClaw 处理任务」，程序使用 **echo 模式**，只验证与商家云的 WebSocket 是否连通。

## 七、命令行启动（可选）

图形界面与命令行共用同一份 `config.yaml`。

```bash
# 默认：使用可执行文件同目录下的 config.yaml，带图形界面
./SaiboAssistant-macOS-arm64

# 指定配置文件路径
./SaiboAssistant-macOS-arm64 -config /绝对路径/config.yaml

# 无图形界面（适合 Linux 服务器 / SSH，须已有完整 config.yaml）
./SaiboAssistant-macOS-arm64 -config ./config.yaml -headless
```

Windows：

```powershell
.\SaiboAssistant-Windows-x64.exe
.\SaiboAssistant-Windows-x64.exe -config C:\path\to\config.yaml
```

## 八、配置完成后如何使用

### 8.1 OpenClaw 任务桥接

1. 本机助手 **已连接商家云**，且已开启 **OpenClaw 通道**。
2. 本机 OpenClaw Gateway 已运行，Token 配置正确。
3. 设备在商家云已绑定到对应 Agent，且 `openclaw_instance_id` 与 Agent 一致。
4. 设备发起 OpenClaw 相关请求后，助手收到 `task.dispatch`，调用本机 OpenClaw，结果回传云端。

### 8.2 听写转发

1. 开启 **听写** 通道并完成 macOS 辅助功能授权（若使用注入）。
2. 在设备上进入听写页，选择「注入」或「悬浮」，点 **开始传输**。
3. 注入模式：先在 Mac/PC 上点好要输入的文本框，再在设备上开始传输；识别文字会写入前台输入框。

### 8.3 保持在线

助手需 **常驻运行**（托盘或后台进程）。关机或退出后，设备侧会显示 Connector 离线，任务无法派发。

## 十、常见问题

| 现象 | 处理 |
|------|------|
| Mac 提示无法打开/来自未知开发者 | 系统设置 → 隐私与安全性 → 仍要打开；或 `xattr -cr ./SaiboAssistant-macOS-arm64` |
| 无法连接商家云 | 检查 `server_url` 是否含 `/bridge/connector`、端口与防火墙、商家云是否开启 `bridge_enabled` |
| 连接码无效 | 确认 6 位数字、未过期；在管理台重新生成 |
| 能连云但收不到任务 | 检查设备 MAC 是否与绑定设备一致；OpenClaw 通道是否开启 |
| OpenClaw 任务失败 | 检查 Gateway 是否运行、`gateway_token` 是否与 `openclaw.json` 一致 |
| macOS 听写不注入 | 系统设置 → 辅助功能 → 允许「赛搏小小助手」/ `openclaw-connector` |
